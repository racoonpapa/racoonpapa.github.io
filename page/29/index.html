다양한 언어 훑어보기 - 준비

보통 백엔드에서 사용하는 각종 언어에 대해 가볍게 훑어보는 포스팅. 심도 있는 주제는 아니고, 자신이 사용하지 않는 언어도 대충 이런 느낌으로 프로그래밍 되는구나를 보면 되겠다. 시작은 아마도 Java가 될 테고, C#, Python, Javascript, Go, Rust 정도를 다루게 될 예정이다. (C++은 코드가 너무 장황해서 뺐다.) REST API 류를 다루다 보면 가장 자주 다루는 부분은 JSON을 해석해서 자신의 데이터 포맷으로 담고, 가공된 이후에 JSON으로 바꾸어 반환하는 부분일 것이다. 딱 이 부분만 구현하는 것을 최대한 간단하게 살펴보겠다. (강좌 아니다) 본격적 코딩 이전에 데이터부터 생성했다. 이건 자바스크립트가 가장 편리하게 만들 수 있.......

Go 언어에 대한 잡담

작년 초에 서버 쪽 일을 시작하면서 사용할 만한 언어들을 물색했는데 메모리, 성능에 민감했고 클라이언트가 C++로 작성되기 때문에 Javascript나 Python은 배제하였고, Java나 C#처럼 특정 런타임을 거치는 언어들도 제외하고 나니 의외로 몇 개 남는 것이 없었다. 물론 C/C++을 선택할 수도 있으나 혼자 하기에는 작업량이 버겁다 보니 '성능이 좋으면서 생산성도 좋은' 언어를 고민하였다. 그러면서 떠올린 게 Go와 Rust였는데 양쪽을 조금씩 써보다가 최종적으론 Go를 사용하기 시작했다. 내 지인들로부터 ‘너무 마이너 한 언어를 선택한 거 아냐?’라는 이야기도 들었는데 1년이 넘게 쓰면서 든 생각이 Go가 다소 애매한 부분도.......
