Gitlab/Github의 Webhooks

오늘 퇴근하다가 팀원들의 커밋 내용들을 한눈에 볼 수 있는(또는 추적할 수 있는) 방법을 만들어야겠다는 생각이 들었다. 물론 git log로 보는 방법도 있고 SourceTree로 보는 방법도 있긴 하지만 내가 재미로 만든 인트라넷에 아예 통합해서 보고 싶었다. 여담이지만 팀원들이 무엇을 하는지 작업 내역이 공유되는 것은 매우 중요하다. 작년에 잠시 화이트보드에 스크럼 보드를 만들기도 했지만 우리에겐 별로 맞지 않는다고 생각했고, Trello면 되려나 싶었지만 트렐로에 일부러 접근을 하는 것이 상당히 불편하다고 생각이 들었다. 이는 마이크로소프트 팀즈에서도 지원되지만 팀즈의 할 일 목록 관리는 매우 불편하고 팀즈 자체가 채팅 이외.......
