InfluxDB + Grafana

뭐 유사한 고민을 해본 사람들을 위해 적어보는 뻘글. 서버 애플리케이션들을 만드는 와중에 모니터링 도구를 만들 계획이 있었다. 이미 있는 솔루션들이 많기는 한데 뭔가 더 내 입맛에 맞는 것을 넣기 위해서 보통 TICK 스택(Telegraf + InfluxDB + Chronograf + Kapacitor)이라고 말하는 것을 사용하지 않고 내 경우엔 그냥 데이터베이스만 InfluxDB를 쓰고 대시보드는 Grafana를 사용하기로 했다. Telegraf 역할을 하는 부분은 직접 작성했고 구동을 원격에서 리포팅을 제어하기 위해서 간단한 api들을 심었다. 이전에 잠깐 InfluxDB를 테스트 삼아 써보긴 했는데 본격적으로 사용하는 것은 처음이고 Grafana 역시 마찬가지였다. InfluxDB 말.......
